## 0. Bottom Line

### Data

- Shared baseline (`baseline_noop`, n=500): **23.4% csv2_yes** (117/500).
- Existing L1-neuron comparator (`l1_neuron`, α=3.0, n=500): **27.4% csv2_yes** (137/500), **+4.0 pp** vs baseline **[+0.6, +7.6]**.
- D7 causal intervention (`causal_locked`, α=4.0, n=500): **14.4% csv2_yes** (72/500), **-9.0 pp** vs baseline **[-12.2, -5.8]**.
- Binary judge moves in the same direction:
  - baseline: 30.0% (150/500)
  - L1: 33.2% (166/500), **+3.2 pp** **[-0.2, +6.6]**
  - causal: 19.4% (97/500), **-10.6 pp** **[-14.0, -7.2]**
- Severity-sensitive metrics also move in the same direction for causal:
  - `C`: **-0.40** **[-0.474, -0.328]**
  - `S`: **-0.456** **[-0.548, -0.366]**
  - `V`: **-0.378** **[-0.444, -0.312]**
  - harmful payload share: **-0.1496** **[-0.1743, -0.1253]**

### Interpretation

The trimmed full-500 confirmatory run supports a real benchmark-level result: **the locked causal head intervention reduces jailbreak severity relative to a shared no-op baseline, while the existing L1-neuron comparator makes it worse**. That part survives scrutiny.

What does **not** survive scrutiny is the stronger story that D7 is now fully closed as a selector-specific, publication-ready claim. This full-500 run was deliberately trimmed: the probe family was not completed, and the planned random-head negative control was skipped. So the justified claim is:

> **On this benchmark and generation surface, the locked causal intervention beats both no-op and the current L1-neuron comparator.**

Not yet justified:

> **Gradient-based selection is uniquely responsible for the gain** or **the causal head set is specific rather than a generic matched-K perturbation.**

## 1. Scope Clarification

This file audits the **trimmed** full-500 comparison launched by [`scripts/infra/d7_causal_pilot_trimmed.sh`](../../scripts/infra/d7_causal_pilot_trimmed.sh), not the originally broader D7 plan.

The trimmed comparison contains three completed, judged, CSV2-scored conditions:

1. `baseline_noop`
2. `l1_neuron`
3. `causal_locked`

It intentionally skipped:

1. `probe_locked`
2. `causal_random_head` negative controls

That choice reduced cost and answered the practical "does the causal intervention beat the incumbent baseline?" question. It did **not** satisfy the stronger selector-specificity standard in [measurement-blueprint.md](../measurement-blueprint.md), which requires a negative control for each new intervention family.

2. a broad class of matched-K decode-only head perturbations at this alpha would also reduce `csv2_yes`.

The pilot probe-null is informative, but it is **not** a selector-specific negative control.

## 4. What Withstands Scrutiny

### 4.1 The causal intervention beats both no-op and the current L1 comparator on the primary metric

This is the strongest claim in the file, because it is directly measured on the benchmark, uses the shared baseline, and has a narrow enough CI to be stable at the sign level.

Think of the baseline as the untreated group, L1 as the incumbent treatment, and causal as the challenger. On the primary endpoint, L1 moves the wrong way; causal moves the right way; the gap is not a rounding error.

### 4.2 The causal gain is not explained away by token-cap degeneration

Token-cap hits are real for causal: **112/500** (22.4%). That is a quality concern and must be reported.

But the data do **not** support the story that truncation is creating a false safety win:

- among cap-hit causal rows, **97/112** are `csv2_no`
- cap-hit rows have mean harmful payload share **0.0192**
- non-cap rows have mean harmful payload share **0.1274**
- paired causal effect remains on the non-cap subset: **-9.8 pp** [-13.7, -5.9]
- even the cap-hit subset still goes in the same direction: **-6.3 pp** [-11.6, -0.9]

So the degeneration behaves more like **verbose refusal drift** than **hidden harmfulness being chopped off before scoring**.

### 4.3 The pilot's broad directional result survives, but its finer-grained story changes

The pilot said "causal good, probe null." The full-500 data do not contradict that.

But some pilot details were unstable:

- template concentration changed materially
- the confirmatory run is against L1 + baseline, not probe
- the strongest claim now is not "causal beats probe at full scale" but "causal beats baseline and L1 at full scale"

That is exactly why the larger confirmatory run was worth doing.

## 5. What Does Not Yet Withstand Scrutiny

### 5.1 Selector-specificity is still unproven

The D7 intervention family lacks the control required to say "these particular gradient-ranked heads matter, as opposed to any matched-K perturbation." In AI safety terms, this is the difference between:

- observing a useful behavioral effect, and
- demonstrating that the effect is mechanistically specific rather than a broad distribution shift.

The repo's own measurement contract is right to demand this control. Right now D7 does not meet that bar.

### 5.2 The full-500 run is confirmatory only in the trimmed sense

The original D7 orchestration planned:

1. baseline
2. L1
3. probe
4. causal
5. random-head controls

The trimmed execution kept only 1, 2, and 4. That is enough to answer a practical comparator question. It is not enough to close the full scientific question that D7 was meant to answer.

### 5.3 Capability and harmlessness costs remain unmeasured

The causal intervention clearly changes response behavior:

- mean response length rises from **5739.9** chars at baseline to **8204.8**
- 22.4% of responses hit the generation cap

That is a visible warning sign, but not a substitute for the required capability mini-battery. We still do not know the cost on:

1. unrelated factual accuracy
2. harmless instruction following
3. a light fluency / language-quality proxy

Without those, the intervention looks promising as a benchmark-local mitigation, not yet as a balanced safety improvement.

## 6. Interpretation

This section is intentionally interpretive rather than purely descriptive.

### 6.1 Best current reading

The causal head intervention appears to be doing something real and useful on this refusal/jailbreak surface. The evidence is stronger than the pilot because:

- the n is now 500 rather than 100
- the comparison uses a shared baseline
- the improvement replicates across strict (`csv2_yes`), diagnostic (binary judge), and severity-sensitive (payload share, `C/S/V`) metrics

If the project needed to choose between shipping the incumbent L1 jailbreak intervention or the locked causal D7 intervention **on this exact benchmark surface**, the data say pick the causal one.

### 6.2 Best skeptical reading

The stronger theoretical claim is still under-supported.

A good analogy is drug testing:

- this run is enough to say the new drug outperformed placebo and outperformed the old drug on the main symptom measure;
- it is **not** enough to say the active ingredient is known, because the mechanism-isolating control arm was never run.

That is where D7 stands today.

### 6.3 How the degeneration should be interpreted

The causal intervention is not "clean." It lowers harmfulness while also pushing a meaningful fraction of outputs into long, cap-hitting refusals or rambling safe responses.

That matters in two opposite ways:

- **good news:** the main safety gain does not appear to be a truncation illusion
- **bad news:** the intervention may be paying for some of its gain by making the model behave oddly

So the practical summary is not "great mitigation"; it is "promising mitigation with visible quality debt."

## 7. Most Valuable Next Steps

1. **Run the D7 random-head negative control.**
   This is the single highest-value next step. Without it, D7 remains a strong benchmark result but a weak mechanistic claim.

2. **Either finish `probe_locked` or explicitly archive it as intentionally dropped.**
   Leaving an interrupted partial branch in the canonical directory is a recipe for future confusion.

3. **Add the minimum capability mini-battery for `causal_locked` at α=4.0.**
   Even a narrow three-slice audit would separate "safer" from "safer because the model gets weird and evasive."

4. **If D7 is cited in synthesis, cite this report plus the structured JSON, not the pilot markdown.**
   One file should own the interpretation; one file should own the row-aggregated data.
