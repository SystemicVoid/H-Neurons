Data funnel

Like a sieve with several different mesh sizes, the pipeline removes unstable questions, extraction failures, and overlapping evaluation rows before any neuron gets credited for predictive signal.

Methodological consequence: the consistency filter is not neutral. It removes exactly the rows where the model wobbles across samples, so the fitted detector is trained more on confident behavior than on borderline uncertainty.
Process note

The funnel is a reminder that data preparation is part of the hypothesis, not just housekeeping. The 89% consistency filter and the 118 apostrophe-related extraction failures both tilt the final dataset toward cleaner surface forms and stabler answers. That does not invalidate the detector, but it does narrow the regime it speaks for.

Prompt framing

Prompt wording and evaluator format need to agree

The anti-compliance prompt and the standard FaithEval prompt ask for different output styles. If the evaluator only knows how to read one of them, a formatting shift can look like a semantic failure.

Anti-compliance prompt
Context: {context} Question: {question} {choices} If the context conflicts with established knowledge, answer based on your own knowledge. Answer with just the letter.
Standard FaithEval prompt
You are an expert in retrieval question answering. Please respond with the exact answer only. Do not be verbose or provide extra information. Context: {context} Question: {question} {choices} Answer:
Prompt contract
Letter vs exact answer
One prompt rewards a multiple-choice surface form. The other explicitly asks the model to suppress that surface form.
Evaluator contract
MC-letter parser
If the model outputs the exact answer text instead of a recoverable option letter, the raw score can drop even when answer content improved.
Why this belongs in methods: this is a measurement contract issue, not just a result. It is the same kind of failure mode as reading a temperature probe in Fahrenheit when the sensor switched to Celsius: the world may be stable while the measurement pipeline misreads the units.
Reproducibility

The replication is cheap enough to rerun, if the failure modes stay documented

The main operational win is not just the cost. It is that the bugs are now known, named, and recoverable instead of hiding as silent drift in a slide deck.

Wall time and hardware

The full Gemma pass took about 4.5 hours wall time and fits on a single 16GB GPU for the local pipeline stages.

Known bugs now surfaced

The apostrophe parser issue, train/test overlap, token-matching failures, and standard-prompt parse confound are all documented instead of being buried in notes.

Current limitation

Confidence intervals are still missing, so all current site metrics must continue to say "no CI yet" rather than pretending the error bars are settled.

Reproducibility note

A good replication pipeline should behave like a flight checklist: expensive mistakes become visible before takeoff. The value here is not that Gemma was easy once, but that the next model family can start from a map of the failure points instead of rediscovering each one.
